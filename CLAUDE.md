# CLAUDE.md — bum.bike Project Context

## Project

**Bécyks Unis Montréal / Bikes Unite Montreal** — community bike shop network website for Montreal.
- Live site: https://bum.bike
- GitHub repo: `ateliers-velo/website`

## Stack

- **Eleventy v3** + **Nunjucks** templates
- Deployed to **GitHub Pages** via **GitHub Actions** (`.github/workflows/build.yml`)
- Build command: `npm run build:prod` → output to `public/`
- Dev server: `npm start`
- Trigger: push to `main` branch

## Content Structure

| Type | Location | Layout | Tag |
|------|----------|--------|-----|
| Blog posts / events | `src/content/posts/` | `post.html` | `posts` |
| Bike shop pages | `src/content/ateliers/` | `basic.html` | `ateliers` |
| Static pages | `src/content/pages/` | varies | `nav-items`, `index-pages` |
| Landing pages | `src/content/index/` | `landing.html` | — |

Default tags and layouts per content type are set in directory-level `.json` files (e.g., `src/content/posts/posts.json`). **Do not duplicate these in individual file frontmatter.**

## Bilingual Architecture

- Two languages: **French** (default) and **English**
- Language detected from filename suffix: `filename.fr.md` → FR, `filename.en.md` → EN
- Files without a suffix default to French (`defaultLang: fr` in `src/_data/site.json`)
- Permalink pattern: `/{lang}/{folderName}/{titleSlug}/`
- `translationKey` auto-generated by `eleventyComputed.js` when exactly **2 files share a folder**
  - Both FR + EN versions must live in the same subfolder to be linked as translations
  - The language switcher depends on this pairing

## Key Data Files

- `src/_data/site.json` — site title, description, languages list — **do not change `languages` array length** without updating the translationKey logic
- `src/_data/eleventyComputed.js` — computes `lang`, `permalink`, `translationKey`, `title`, `folderName`, `titleSlug` from filenames and directory structure
- `src/_data/ateliers.json` — GeoJSON FeatureCollection of 20 bike shops (coordinates, logos, URLs); used by the MapLibre map and passed through to `public/` as-is

## CMS Admin Panel

Sveltia CMS (headless, Git-backed) lives at `/admin/`:
- URL: https://bum.bike/admin/
- Auth: GitHub OAuth PKCE (requires GitHub repo write access)
- Config: `src/admin/config.yml`
- Saves content by committing directly to `main`, triggering the normal build

### Bilingual post workflow in CMS
1. Create FR post in "Articles (Français)" → choose a folder slug
2. Create EN post in "Posts (English)" → **use the same folder slug** so Eleventy pairs them as translations
3. Do NOT add `tags` or `layout` to post frontmatter — inherited from `posts.json`

## Important Conventions

- Posts are organized in per-event subfolders (e.g., `src/content/posts/my-event/Post Title.fr.md`)
- Atelier pages with `permalink: false` are redirect stubs pointing to external shop URLs
- The `<!--section-->` separator in content splits pages into `main` and `sidenote` sections (via Nunjucks filters)
- Analytics: GoatCounter at `ateliers-velo.goatcounter.com`
